# Messages Feature with Socket.IO

## Overview
Add real-time messaging between buyers and sellers about listings. Socket.IO for live message delivery, MongoDB for persistence.

## Dependencies to Install
```
socket.io          (server)
socket.io-client   (client)
```

## Architecture

### Custom Server (`server.ts`)
Socket.IO needs a persistent HTTP server. Replace `next dev` / `next start` with a custom server that attaches both Next.js and Socket.IO to the same HTTP server.

- `server.ts` at project root ‚Äî creates `http.createServer`, attaches Next.js request handler, attaches Socket.IO
- Socket.IO server authenticates connections by reading `session-token` cookie
- Each user joins a personal room `user:<userId>` for targeted message delivery

### Socket.IO Events
| Event | Direction | Payload | Description |
|-------|-----------|---------|-------------|
| `connection` | client‚Üíserver | cookie | Auth handshake, join user room |
| `join-conversation` | client‚Üíserver | `{ conversationId }` | Join a chat room |
| `leave-conversation` | client‚Üíserver | `{ conversationId }` | Leave a chat room |
| `send-message` | client‚Üíserver | `{ conversationId, content }` | Send a message |
| `new-message` | server‚Üíclient | `Message` | Broadcast to conversation participants |
| `message-sent` | server‚Üíclient | `Message` | Confirm to sender |
| `typing` | client‚Üíserver | `{ conversationId }` | Typing indicator |
| `user-typing` | server‚Üíclient | `{ conversationId, userId }` | Broadcast typing |
| `mark-read` | client‚Üíserver | `{ conversationId }` | Mark messages as read |

## MongoDB Models

### Conversation
```typescript
{
  participants: [ObjectId ‚Üí User] (required, index, array of exactly 2),
  listing: ObjectId ‚Üí Listing (optional),
  lastMessage: ObjectId ‚Üí Message (optional),
  lastMessageAt: Date (index, for sorting),
  createdAt, updatedAt (timestamps)
}
```
Compound unique index: `{ participants: 1 }` to prevent duplicate conversations.

### Message
```typescript
{
  conversation: ObjectId ‚Üí Conversation (required, index),
  sender: ObjectId ‚Üí User (required),
  content: String (required),
  isRead: Boolean (default: false, index),
  createdAt: Date (index)
}
```
Index on `{ conversation: 1, createdAt: -1 }` for efficient chat history queries.
